Developing¶

The intention is to make sure that all the development workflows, such as building the docs, running the tests and creating the distribution are covered by Paver. The commands are detailed below and can be executed by, for example, typing:

paver docs

To run the tests, first start up a Kyoto Tycoon instance on port 1977 with an index name like test.kct:

paver kyototycoon -p 1977 -f test.kct

and then in another terminal window run:

paver test

Commands¶

pavement.test(func)¶

Build test enviroment and run tests

pavement.kyototycoon(func)¶

Run Kyoto Tycoon server

pavement.sdist(func)¶

Overrides sdist to make sure that our docs and setup.py are generated.

pavement.docs(func)¶

Build environment for creating docs and then build the docs

pavement.github_docs(func)¶

Build the docs and upload to GitHub - You’ll need to be a committer!

pavement.examples(func)¶

Build examples environment

pavement.legislation(func)¶

Runs legislation example
